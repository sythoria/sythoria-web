# Product

## Register

brand

## Users
Developers, power users, and privacy-conscious individuals who want to interact with multiple AI models (OpenAI, Anthropic, Gemini, Ollama, etc.) securely using their own API keys without trusting a third-party web interface.

## Product Purpose
Sythoria-web serves as the official marketing landing page and comprehensive documentation portal for the Sythoria desktop application. Its purpose is to showcase the advantages of the desktop app (privacy-first, local keys, statelessness) and guide users to download the latest desktop version from GitHub.

## Brand Personality
- Tone: Sleek, developer-centric, minimal, clean, trustworthy, and precise.
- 3-word personality: Privacy-focused, Refined, Developer-centric.
- Emotional goals: Confidence in security, appreciation for lightweight and clean design, ease of getting started.

## Anti-references
- Over-saturated, generic warm-cream/beige SaaS landing pages (cookie-cutter "AI workflow tools").
- Overstimulating visual clutter, such as floating decorative glass cards or flashy, meaningless gradients.
- Generic AI tropes like side-stripe border accents on cards, tiny tracked uppercase kicker eyebrows above every single heading, and numbered section eyebrows (01, 02, 03) when there is no sequence.
- Gradient text as a main layout decoration (e.g. `background-clip: text` combined with gradients used as a generic aesthetic fix).

## Design Principles
1. **Focus on the Download (Action-Oriented)**: Every page, link, and call-to-action must logically funnel the user toward downloading the desktop app.
2. **Stateless Privacy Reassurance**: The design should evoke transparency and safety, mirroring the desktop application's commitment to not storing keys or logs.
3. **Typographic Restraint & Clean Contrast**: Leverage clean sans-serif typography paired with precise monospaced elements to appeal to developers, maintaining high legibility over decorative clutter.
4. **Predictable and Cohesive Patterns**: Ensure both the landing page and the MDX documentation viewer feel like a unified product through shared token variables and spacing.

## Accessibility & Inclusion
- Contrast: Maintain strict WCAG AA standards (>= 4.5:1 for body and placeholder text, >= 3:1 for large text).
- Motion: Respect `@media (prefers-reduced-motion: reduce)` by providing transitions or immediate states.
- Usability: Fully accessible keyboard navigation and clearly labeled interactive states.
