# Streaming Responses

Sythoria leverages high-performance **Server-Sent Events (SSE)** to stream model outputs token by token. Instead of waiting for a complete model response, which can introduce seconds of perceived latency, text appears in the user interface the millisecond it is generated by the remote or local API provider.

---

## Architectural Flow

The diagram below illustrates how a streaming connection is negotiated, parsed, and rendered:

```
┌───────────┐           ┌────────────────────┐           ┌───────────────────┐
│  HTML5    │           │    Tauri / Rust    │           │    AI Provider    │
│  Chat UI  │           │    App Core        │           │    API Endpoint   │
└─────┬─────┘           └─────────┬──────────┘           └─────────┬─────────┘
      │                           │                                │
      │  1. Submit Message        │                                │
      ├──────────────────────────►│  2. Open HTTPS Connection       │
      │                           ├───────────────────────────────►│
      │                           │                                │
      │                           │  3. Stream SSE Frames          │
      │                           │◄───────────────────────────────┤
      │  4. Send Token Buffer     │                                │
      │◄──────────────────────────┤                                │
      │  (requestAnimationFrame)  │                                │
      │                           │                                │
      │                           │  5. Send [DONE] Signal         │
      │                           │◄───────────────────────────────┤
      │  6. Stream Complete       │                                │
      │◄──────────────────────────┤                                │
```

1. **Initiation**: The user submits a prompt. The React UI forwards the payload to the Tauri Rust core.
2. **HTTP Channel Negotiation**: The Rust core initiates an HTTP client request to the provider using HTTP/2 or HTTP/3 multiplexing. The headers request `Accept: text/event-stream` and `Cache-Control: no-cache`.
3. **Chunk Parsing**: As network TCP packets arrive, the Rust core reads and parses incoming bytes on a background thread. It extracts the raw `data: ` frames, strips protocol boilerplate, and validates JSON blocks.
4. **UI Thread Deserialization**: Parsed token updates are forwarded across the Tauri IPC channel to the frontend, which groups tokens into a rendering buffer.

## Protocol Event Frames

Different API providers structure their stream events in slightly different formats. Sythoria's Rust core maps these various schemas into standardized event streams.

### 1. Standard Token Frame (OpenAI/Compatible)

```http
data: {"id":"chatcmpl-123","object":"chat.completion.chunk","created":1700000000,"model":"gpt-4o","choices":[{"index":0,"delta":{"content":" Hello"},"finish_reason":null}]}
```

### 2. Thinking / Reasoning Block Frame (DeepSeek / Anthropic)

For models that output thinking paths (like DeepSeek-R1 or Claude 3.7 Sonnet), reasoning steps are streamed via specific tags or block types. Sythoria separates these into a dedicated collapsible UI panel:

```http
data: {"id":"chatcmpl-456","object":"chat.completion.chunk","choices":[{"index":0,"delta":{"reasoning_content":"Searching workspace for calculation rules..."},"finish_reason":null}]}
```

### 3. Tool Call Frame

When the model invokes a tool, it streams the name and arguments incrementally:

```http
data: {"id":"chatcmpl-789","choices":[{"delta":{"tool_calls":[{"index":0,"id":"call_abc","function":{"name":"project_grep","arguments":"{\n  \"que"}}}]}]}
```

## Real-Time UI Rendering Mechanics

Streaming text at speeds exceeding $100$ tokens per second (typical of modern frontier models) can easily choke web rendering threads. Sythoria implements specific performance optimizations to ensure smooth UI performance:

- **Chunk Buffering**: Rather than triggering a React state re-render for every incoming character, Sythoria collects characters in a micro-buffer and flushes them to the UI state using a scheduled `requestAnimationFrame` loop (running at 60fps/120fps).
- **Incremental Markdown Compilation**: The UI parses and displays Markdown on the fly. Code blocks remain as raw text until a code fence is closed, at which point Prism/Shiki highlights the code syntax.
- **Math Typesetting (LaTeX)**: Mathematical expressions inside standard delimiters (e.g., `$...$` and `$$...$$`) are buffered. LaTeX is typeset using KaTeX only after the expression's boundary tags are fully received, avoiding partial-equation rendering errors.
- **Backpressure Management**: If the browser tab loses focus or the main thread lags, Sythoria automatically coalesces tokens into larger blocks to catch up, ensuring the UI remains responsive.

## Error Handling & Stream Reconnection

If a stream is interrupted due to a network drop, Sythoria attempts to recover without losing context:

1. **Mid-Stream Drop Recovery**: If the connection drops before the provider sends a `finish_reason`, Sythoria holds the current output and attempts to reopen the stream with an offset prompt.
2. **Finish Reason Handlers**: If the stream terminates due to context length limits (`finish_reason: "length"`), Sythoria alerts the user and offers a quick action to summarize the conversation or prune older messages.
3. **Manual Interrupt**: Clicking the **Stop** button or pressing `Escape` immediately aborts the active HTTP request, terminating the socket connection while preserving the generated output.

## Troubleshooting

### 1. Responses Render All at Once (No Streaming)

- **Cause**: There is a proxy, VPN, or security software (such as Cloudflare Warp or local antivirus) buffering HTTP responses on your machine.
- **Solution**:
  - Verify that you do not have middleware buffering network requests.
  - If using a self-hosted provider (like Ollama or LocalAI), ensure that the backend does not have buffer settings active in its configuration.

### 2. UI Lags or Freezes During High-Speed Output

- **Cause**: High-throughput models generating text faster than the DOM can update (common when rendering large tables or detailed code files).
- **Solution**:
  - Enable **Simplify Markdown Formatting during Stream** in Settings. This disables real-time highlighting and LaTeX parsing until generation is complete.
  - Reduce your monitor's refresh rate override in the app's advanced hardware settings.

### 3. Truncated Streams with `ERR_CONNECTION_RESET`

- **Cause**: API rate limits or network timeout settings from the provider.
- **Solution**:
  - Check the developer console logs for rate-limiting status codes (HTTP `429`).
  - In Settings, increase the **HTTP Connection Timeout** parameter (default: 30 seconds) to prevent the client from dropping slow connections.
