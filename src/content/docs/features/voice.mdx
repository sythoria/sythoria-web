# Voice Input & Realtime Audio

Sythoria offers robust voice capabilities, supporting both **offline speech-to-text dictation** and **low-latency, bi-directional realtime voice conversations**.

Whether you want to transcribe local voice notes securely or engage in hands-free live discussion, Sythoria routes audio streams with privacy-first standards.

---

## Local Whisper Engine (Offline Dictation)

For standard text dictation, Sythoria uses a local, native port of `whisper.cpp` compiled into the Tauri/Rust core application. Your voice is captured, processed, and transcribed entirely on your local CPU or GPU. No audio data ever leaves your machine.

### How Offline Transcription Works

<VoiceTranscriptionFlow />

1. **Microphone Capture**: Clicking the microphone icon initiates audio capture from your default system input device.
2. **Voice Activity Detection (VAD)**: Sythoria runs a lightweight local VAD pass (powered by WebAssembly Silero VAD) to determine when you begin and end speaking.
3. **Resampling**: The raw audio stream is normalized and resampled to mono 16kHz, 16-bit WAV format (the target input expected by Whisper).
4. **Whisper Transcription**: The processed buffer is loaded into the local GGML model context, which transcribes the audio and injects the text directly into the chat input bar.

## Setup & Model Installation

Before using local dictation, you must download a Whisper model file:

1. Navigate to **Settings > Voice / Whisper**.
2. Turn on **Enable Voice Input**.
3. Select a model preset from the dropdown (or select _Custom Model_ to load your own `.bin` GGML file).
4. Click **Download Model**. The application downloads the selected model securely from Hugging Face and saves it in the local app directory.
5. Once the download completes, the microphone button will appear in the chat input toolbar.

## Hardware Acceleration

To optimize local inference times, Sythoria utilizes hardware-specific acceleration backends via `whisper.cpp`:

- **macOS (Apple Silicon)**: Utilizes Apple's **CoreML** framework and **Metal Shaders** to run models directly on the Apple Neural Engine (ANE) or GPU, keeping CPU usage minimal.
- **Windows & Linux (NVIDIA)**: Leverages **CUDA** acceleration. If CUDA runtimes are detected on startup, Whisper offloads decoding layers to the GPU.
- **Windows & Linux (AMD)**: Utilizes **ROCm / HIP** acceleration.
- **Intel Devices**: Integrates **OpenVINO** for CPU/iGPU acceleration.

You can select your preferred backend under **Settings > Voice > Hardware Acceleration Backend**.

## Realtime / Live Voice Mode (Realtime APIs)

In addition to offline dictation, Sythoria supports **bidirectional voice mode** for interactive conversations. This bypasses text generation entirely and opens a low-latency audio connection with compatible API providers.

<RealtimeAudioFlow />

### Supported Realtime Protocols

1. **OpenAI Realtime API**: Uses WebSockets to stream 24kHz audio packets back and forth, allowing you to speak to models like `gpt-4o-realtime` and receive natural verbal responses with sub-second latency.
2. **Gemini Multimodal Live API**: Connects via WebSockets using bidirectional audio streaming to support live conversations with Google Gemini models.

### Setup and Usage

1. Go to **Settings > Voice > Realtime Mode**.
2. Select your provider and choose an audio-capable model (e.g., `gpt-4o-realtime-preview` or `gemini-2.0-flash-exp`).
3. Click the **Phone / Live Voice** icon next to the input field.
4. Sythoria will establish a connection, request microphone access, and play a sound indicating it is listening. You can speak naturally; the model will interrupt its response if you speak while it is talking.

## Troubleshooting

### 1. "Microphone Access Denied" or Permission Errors

- **Symptom**: Clicking the record button shows an error dialog or fails to register audio waves.
- **Resolution**:
  - **macOS**: Go to _System Settings > Privacy & Security > Microphone_ and ensure Sythoria is checked.
  - **Windows**: Go to _Settings > Privacy & security > Microphone_ and verify that desktop apps are allowed to access your microphone.
  - **Linux**: Ensure your user is part of the `audio` group (e.g., `sudo usermod -aG audio $USER`), and check PulseAudio/PipeWire permission controls.

### 2. Slow Dictation & Transcription Lag

- **Symptom**: Transcription takes 10+ seconds to appear after you stop speaking.
- **Resolution**:
  - Check your model size. If you are using `large-v3-turbo` on a standard CPU, switch to `base.en` or `distil-small.en`.
  - Ensure hardware acceleration is enabled. Check **Settings > Voice > Backend Status** to verify that CUDA or CoreML is active.

### 3. Missing Shared Library / DLL Load Errors (CUDA)

- **Symptom**: Console logs show `Failed to load CUDA runtime dll` or fallback to CPU on Windows.
- **Resolution**: Ensure you have the compatible CUDA Toolkit version installed (minimum version 12.1 recommended) and that the CUDA path is added to your environment `PATH` variable.

### 4. Background Noise & Empty Transcripts

- **Symptom**: Whisper transcribes random silent rooms as repeating words (e.g. "you", "thanks for watching").
- **Resolution**: Adjust the **Silence Threshold (VAD)** in Settings. Increasing this value prevents faint background noises or keyboard clicks from triggering the audio transcription loop.
