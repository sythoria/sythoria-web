# Multi-Provider Chat

Sythoria is engineered to unify different AI providers—both cloud-based frontier APIs and locally-hosted engines—under a single, cohesive user interface. Instead of switching tabs, applications, or accounts, Sythoria allows you to interact with multiple models, compare their reasoning outputs, and transition between them fluidly within the same conversation thread.

---

## The Power of Mid-Conversation Model Switching

Use the **model dropdown** in the chat input bar to change models at any point, even between subsequent messages in a single chat. This enables powerful hybrid workflows:

- **Iterative Debugging**: Start an architectural discussion with a highly capable reasoning model like `claude-fable-5`, then switch to a local `llama4:maverick` via Ollama for standard boilerplate generation.
- **Multimodal Visual Inspection**: Attach layout screenshots using the [Appshots tool](/docs/features/appshots) to a vision-enabled model like `gpt-5.6-sol`, then pivot to `gemini-3.5-flash` for high-context data parsing.
- **Verification & Comparison**: Query a frontier reasoning model, then switch to a local offline model to verify structural execution without uploading proprietary results.

> [!TIP]
> Switching models is immediate. The Sythoria interface manages provider handshakes behind the scenes, so there is no lag or connection delay when changing models.

## Context Translation Engine

Because different AI providers utilize distinct input formats and payload structures (such as Anthropic’s Messages API structure versus OpenAI’s Chat Completions JSON schema), Sythoria features a built-in **Context Translation Engine**.

When switching models, Sythoria dynamically parses the active chat thread and compiles it into the schema required by the destination provider:

- **System Prompts**: Automatically injected at the top-level parameter for Anthropic or converted into a `system` role message for OpenAI-compatible endpoints.
- **Multi-Modal Data**: Screenshots and text attachments are serialized into base64 image block structures or text arrays depending on provider vision capabilities.
- **Unsupported Roles**: System, User, and Assistant message alternates are consolidated to prevent structure errors (e.g., merging consecutive messages of the same role).

## Local-First Security Design

Your API keys are highly sensitive. Unlike SaaS interfaces that route your keys through their own proxy servers, Sythoria is a desktop client that communicates **directly** with the provider endpoints:

1. **Secure Storage**: Credentials are saved using your operating system's native secure storage layer (macOS Keychain, Windows Credential Manager, or Linux Secret Service via DBus/libsecret).
2. **Direct HTTPS Traffic**: Requests are sent over TLS directly from your machine to the provider’s URL.
3. **No Intermediate Logging**: Sythoria does not log prompts, tokens, or API keys to any telemetry server. Your conversation history is stored locally in an encrypted SQLite database on your device.

## Connection Lifecycle Indicators

Each model configuration displays a color-coded status indicator next to the model selector, showing the status of the connection in real-time:

- 🟢 **Green (Connected)**: A background heartbeat request succeeded. The endpoint is active and authenticated.
- 🟡 **Yellow (Validating)**: Handshaking or testing credentials.
- 🔴 **Red (Offline / Error)**: Connection failed. Click the indicator to view the error log (e.g., `401 Unauthorized` or `ERR_CONNECTION_REFUSED`).

## Configuring Custom Providers

For custom endpoints (Together AI, Groq, Fireworks AI, vLLM, or LM Studio), choose **Custom** from the provider dropdown and configure the following parameters:

- **API Base URL**: The endpoint URL (e.g., `https://api.groq.com/openai/v1` or `http://localhost:1234/v1`).
- **API Key**: The credentials required by the server (leave blank if running locally without authentication).
- **Model ID**: The string identifier matching the model hosted by your custom endpoint.

### Custom Headers and Payload Injection

For advanced setups (such as private enterprise proxies), you can inject custom HTTP headers and custom body parameters in **Settings > Model Settings > Advanced JSON configuration**:

```json
{
  "headers": {
    "X-Organization-ID": "corp-dev-division",
    "X-Routing-Key": "priority-gpu-cluster"
  },
  "custom_body_params": {
    "frequency_penalty": 0.2,
    "repetition_penalty": 1.1
  }
}
```
