# Adaptive Motion Design

Sythoria is built with a premium desktop-grade aesthetic, delivering a fluid, tactile user interface that mirrors the responsiveness of native operating system window managers. Rather than relying on simple linear transitions, Sythoria utilizes a bespoke **motion design system** backed by custom spring-physics equations, hardware-aware performance scaling, and glassmorphic UI components.

---

## Spring-Physics Engine

To mimic the behavior of physical materials, Sythoria replaces traditional cubic-bezier transition timelines with mass-tension-friction (spring-physics) calculations. This prevents animations from feeling robotic, ensuring that panels, modals, and buttons decelerate and settle with real-world momentum.

The core framework exposes three spring tokens:

| Token                | Stiffness (Tension) | Damping (Friction) | Mass  | Typical Use Cases                                                  |
| :------------------- | :------------------ | :----------------- | :---- | :----------------------------------------------------------------- |
| **`springs.snappy`** | `300`               | `30`               | `1.0` | Sidebar toggling, icon state transitions, tooltips, toast entries  |
| **`springs.gentle`** | `120`               | `14`               | `1.0` | Modal windows, settings panel transitions, large bento cards       |
| **`springs.bouncy`** | `400`               | `10`               | `0.8` | Onboarding sequences, welcome banners, dynamic notification alerts |

### Micro-Scaling (Tactile Feedback)

Sythoria applies micro-scaling properties to interactable components, providing immediate visual weight to user actions:

- **Hover (`pop`)**: Scales items outward to `1.04` using `springs.snappy` to indicate interactive focus.
- **Active Click (`press`)**: Compresses controls down to `0.95` scale, giving buttons a physically responsive "depress" sensation.
- **Secondary Actions (`subtle`)**: A minor compression to `0.98` scale, reserved for inline text links and dropdown selectors.

## Glassmorphism & Visual Layering

Sythoria's aesthetic relies on glassmorphic paneling, mimicking sandblasted acrylic overlays. This design relies on precise CSS and Tailwind CSS v4 configurations to maintain readability and visual hierarchy:

```css
/* Core glassmorphic overlay variables in globals.css */
.glass-panel {
  background-color: var(--theme-landing-card);
  backdrop-filter: blur(20px) saturate(180%);
  border: 1px solid rgba(255, 255, 255, 0.08);
  box-shadow: 0 8px 32px 0 rgba(0, 0, 0, 0.37);
}
```

### Ambient Glows & Radial Gradients

The interface utilizes hardware-accelerated radial glows (`--theme-glow-primary` and `--theme-glow-secondary`) that track cursor coordinates on bento-cards. As the mouse glides across a card, a dynamic mask shifts the lighting source, illuminating the underlying glass panel borders.

## Performance-Adaptive Motion Architecture

To ensure high frame rates on energy-saving notebooks and older desktops, Sythoria monitors hardware concurrency and system responsiveness in real-time, scaling down animations as needed.

<MotionAdaptiveFlow />

### 1. Reduced Motion Compliance

Sythoria respects accessibility preferences at the operating system level. If the media query `prefers-reduced-motion: reduce` is detected:

- All spatial translations, spring scales, and offset movements are disabled.
- Transitions default to instantaneous visibility swaps or basic crossfades.
- This behavior can also be toggled manually in the application settings under **Accessibility > Enable Motion**.

### 2. Hardware Concurrency Checks

During initialization, the client evaluates `navigator.hardwareConcurrency`:

- **Low-Core Threshold (4 or fewer threads)**: The client automatically enters **Performance Saver Mode**.
- Dynamic backdrop filters (`backdrop-blur`) and heavy canvas rendering loops (e.g., interactive particle spheres) are replaced with solid, opaque CSS backgrounds to conserve CPU cycles.

### 3. Real-Time FPS Drop Recovery

If the frame rate drops below **45 FPS** for more than 10 consecutive frames:

- Non-essential background animations (such as floating ambient gradients or orbit path animations) are dynamically paused.
- Spring stiffness settings are clamped to simplified, hardware-accelerated CSS keyframe transitions to restore UI fluidity.

## Framer Motion Integration Guide

For developers contributing to Sythoria components, standard layout animations are wrapped with customized Framer Motion configurations.

### Custom Dynamic Button Example

```tsx
import { motion } from "framer-motion";

export const MotionButton = ({ children, onClick }) => {
  return (
    <motion.button
      whileHover={{ scale: 1.04 }}
      whileTap={{ scale: 0.95 }}
      transition={{
        type: "spring",
        stiffness: 300,
        damping: 30,
      }}
      className="px-4 py-2 bg-accent text-text-primary rounded-lg glass-panel"
      onClick={onClick}
    >
      {children}
    </motion.button>
  );
};
```

### Collapsible Layout Panels

To avoid layout thrashing during panel expanding or collapsing (e.g., the Chat History Sidebar), the width container is animated using CSS grid-template-rows or dynamic flex-basis styling. This permits adjacent text content fields to scale gracefully without stuttering or truncating awkwardly during transitions.
