# NVIDIA NIM

NVIDIA NIM (NVIDIA Inference Microservices) provides optimized serverless inference endpoints for state-of-the-art open-source models running on accelerated NVIDIA GPU infrastructure. Get production-grade performance, high throughput, and low latency from models like Llama 4, Gemma 4, Nemotron, Mixtral, and more without managing your own GPU hardware.

## Prerequisites

- An NVIDIA Developer account
- An API key generated from the NVIDIA API Catalog

## Setup

### Using the Serverless Cloud API

1. Go to [build.nvidia.com](https://build.nvidia.com/)
2. Sign in with your NVIDIA account (new developer accounts receive free credits)
3. Click **Get API Key** or generate a new key from your profile dashboard (keys begin with `nvapi-`)
4. In Sythoria, open **Settings** from the sidebar
5. Select **NVIDIA NIM** from the **Provider Preset** dropdown — the API base URL and default model fill in automatically
6. Paste your API key into the **API Key** field
7. The connection indicator should turn **green** within a few seconds

> Your API key is stored securely in your OS keychain. It is never sent to Sythoria's servers. See [Privacy & Security](/docs/privacy).

## Model ID format

NVIDIA NIM uses the `provider/model-name` naming scheme. When entering a model in Sythoria's **Model ID** field, use the exact ID from [build.nvidia.com/explore/discover](https://build.nvidia.com/explore/discover). Examples:

- `meta/llama-4-maverick`
- `meta/llama-3.3-70b-instruct`
- `nvidia/llama-3.3-nemotron-super-49b-v1.5`
- `google/gemma-4-31b-it`

> Ensure you include the provider prefix (e.g., `meta/`, `google/`, `nvidia/`) to avoid API gateway routing errors.

## Running NVIDIA NIM locally

If you are running self-hosted NVIDIA NIM containers on local GPU hardware or a private cloud, you can connect them to Sythoria using a **Custom** provider configuration:

1. Under **Settings**, click **Add Model**
2. Choose **Custom** from the provider dropdown
3. Enter your local server's endpoint in the **API Base URL** field (e.g., `http://localhost:8000/v1/chat/completions` or `http://<your-gpu-server-ip>:8000/v1/chat/completions`)
4. Set the **Model ID** to match the model running inside your container (e.g., `meta/llama3-8b-instruct`)
5. If your container does not require authentication, you can leave the **API Key** field blank

## Troubleshooting

| Error                        | Cause                                                | Fix                                                                                                        |
| :--------------------------- | :--------------------------------------------------- | :--------------------------------------------------------------------------------------------------------- |
| **401 Unauthorized**         | API key is invalid, incorrectly copied, or expired   | Generate a new key at [build.nvidia.com](https://build.nvidia.com/) and ensure it starts with `nvapi-`     |
| **403 Forbidden**            | Missing model licensing terms or account permissions | Log in to the build portal, accept the model's licensing terms, or verify your account level               |
| **404 Model not found**      | Incorrect model ID or deprecated model version       | Use the exact model ID from [build.nvidia.com/explore/discover](https://build.nvidia.com/explore/discover) |
| **429 Rate Limit**           | Exceeded free tier or concurrent usage limits        | Wait a moment and retry, or add a billing method to your NVIDIA developer account                          |
| **503 Service unavailable**  | Model endpoint is scaling up or temporarily down     | Wait a moment and retry — NVIDIA scales serverless endpoints dynamically                                   |
| **504 Gateway Timeout**      | The model endpoint failed to respond in time         | Reduce context length, or retry after peak demand subsides                                                 |
| **Connection indicator red** | Network issue or invalid configuration               | Verify the API base URL is `https://integrate.api.nvidia.com/v1/chat/completions` and click **Refresh**    |
