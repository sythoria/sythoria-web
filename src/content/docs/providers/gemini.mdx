# Google Gemini

Connect Sythoria to Google's Gemini API to chat with Google's most capable models, including the latest Gemini 3.5 series, Gemini 3.1 series, Gemini 2.5 series, Gemini 2.0 series, and Gemini 1.5 series. Gemini models offer industry-leading context windows (up to 2 million tokens) and strong performance on reasoning, coding, and multimodal tasks.

## Prerequisites

- A Google account
- An API key generated from Google AI Studio

## Setup

1. Go to [aistudio.google.com/apikey](https://aistudio.google.com/apikey).
2. Click **Create API Key**, select your project, and copy the key.
3. In Sythoria, open **Settings** from the sidebar.
4. Select **Google Gemini** from the **Provider Preset** dropdown — the API base URL and default model will fill in automatically.
5. Paste your API key into the **API Key** field.
6. The connection indicator should turn **green** within a few seconds, indicating successful authentication.

> [!NOTE]
> Your API key is stored securely in your OS keychain. It is never sent to Sythoria's servers. See [Privacy & Security](/docs/privacy).

## Model Capabilities & Features

### OpenAI-Compatible Endpoint

Sythoria utilizes Google's official **OpenAI compatibility layer** (`/v1beta/openai/chat/completions`). This maps standard OpenAI API requests onto the Gemini backend.

- **Parameters**: Standard parameters like `temperature`, `max_tokens`, `stop`, and `stream` are fully supported.
- **System Instructions**: System messages (developer roles) are correctly translated to direct instructions for the Gemini models.

### Deep Think & Reasoning Modes

The latest frontier models, such as the upcoming `gemini-3.5-pro` and `gemini-3-deep-think` (as well as `gemini-2.0-flash-thinking-exp`), incorporate an internal reasoning step.

- **Thinking Traces**: During a chat, the model produces a hidden or visible thought process before generating the final response.
- **Use Cases**: Best suited for competitive programming, advanced mathematical proofs, and complex debugging scenarios.

### Multimodal Inputs

All Gemini models are natively multimodal. You can attach images, diagrams, or screenshots to your messages. Sythoria encodes these assets and transmits them directly through the OpenAI compatibility layer, allowing you to ask questions about visual data.

### Switching Models Mid-Conversation

Sythoria allows you to switch between different Gemini models dynamically. For instance, you can use the fast `gemini-3.5-flash` for initial discussion and switch to `gemini-3.1-pro` (or the upcoming `gemini-3.5-pro` once available) when requesting a complex code implementation or deep analytical review.

## Pricing

Google AI Studio provides a highly generous **Free Tier** for developers, allowing you to build and test applications with rate limits (typically 15 RPM for Flash models).
For production workloads, you can enable billing in your Google Cloud Project to pay per token. See [ai.google.dev/pricing](https://ai.google.dev/pricing) for the latest pricing schedules.

## Troubleshooting

| Error                                   | Cause                                                                                            | Fix                                                                                                                                            |
| :-------------------------------------- | :----------------------------------------------------------------------------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------- |
| **400 Invalid Argument**                | Request contains unsupported parameters or syntax errors.                                        | Verify the request format and check if your settings have any non-standard parameters enabled.                                                 |
| **400 Failed Precondition**             | Free tier is unavailable in your current geographic region.                                      | Go to Google AI Studio, link a Google Cloud Project, and enable billing to access models from restricted locations.                            |
| **401 Unauthorized**                    | Your API key is incorrect, revoked, or expired.                                                  | Generate a new API key at [aistudio.google.com/apikey](https://aistudio.google.com/apikey) and paste it into Settings.                         |
| **403 Forbidden**                       | The Generative Language API is not enabled in your Google Cloud project, or your key is blocked. | Verify that the Generative Language API is enabled in your Google Cloud Console. Check for IP/referrer restrictions.                           |
| **404 Model Not Found**                 | The specified model ID is incorrect, or the model has been retired.                              | Double-check that your model ID matches the table above exactly. Do not use shut-down model versions (e.g. Gemini 2.0 Flash).                  |
| **429 Rate Limit / Resource Exhausted** | You have exceeded the free tier quota limits or billing limits.                                  | Wait a few minutes before retrying. To increase quotas, upgrade your Google AI Studio account to a paid plan.                                  |
| **Connection indicator red**            | Network offline, firewall blocking Google domain, or incorrect API Base URL.                     | Verify your internet connection. Ensure the API Base URL is set to `https://generativelanguage.googleapis.com/v1beta/openai/chat/completions`. |
